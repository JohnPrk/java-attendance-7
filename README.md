# java-attendance-precourse

## ⚙️ 구현 순서

### 1단계 - 파일 입출력 및 크루원 출석 기록 저장

- [x] 데이터를 저장할 핵심 도메인 객체(`Crew`, `Attendance`) 및 일급 컬렉션(`Crews`, `Attendances`) 정의
- [x] `attendances.csv` 파일을 Files.readAllLines()로 읽어와 파싱
- [x] 읽어온 데이터를 도메인 객체에 매핑하여 저장(`초기 데이터 적재`)

### 2단계 - 메뉴 입력 구현
- [x] 메뉴 도메인 객체(`Menu`-Enum) 정의
- [x] 메뉴 입력 요청 메시지 출력
- [x] [사용자] 메뉴 입력
- [x] 메뉴 입력에 따른 유효성 검사
- [x] 사용자 입력 유효성 검사 실패시 재입력 반복 로직 추가 <=개발과정에서 추가

### 3단계 - 출석 등록 구현
- [x] 주말, 공휴일 객체 정의
- [x] 주말 또는 공휴일 유효성 검사
- [x] 크루원 이름 입력 요청 메시지 출력
- [x] [사용자] 이름 입력
  - 입력 유효성 검사
  - 외부 유효성 검사(초기 데이터에 크루원이 있는지 검증)
- [x] 사용자 출석 시간 입력 요청 메시지 출력
- [x] [사용자] 출석 시간 입력
  - 입력 유효성 검사
  - 도메인 유효성 검사
- [x] 출석 등록 완료 메시지 출력

### 4단계 - 출석 수정 구현
- [x] 크루원 이름 입력 요청 메시지 출력
- [x] [사용자] 이름 입력
  - 기존의 유효성 검사 사용
- [x] 수정할 날짜(일) 입력 요청 메시지 출력
- [x] [사용자] 수정할 날짜(일) 입력
  - 입력 유효성 검사
  - 도메인 유효성 검사
- [x] 수정할 시간 입력 요청 메시지 출력
- [x] [사용자] 수정할 시간 입력
  - 기존의 유효성 검사 사용
- [x] 출석 수정 완료 메시지 출력

### 5단계 - 출석 기록 조회
- [x] 전체 출석 기록 도메인 정의
- [x] 제적 여부 확인 도메인 정의
- [x] 크루원 이름 입력 요청 메시지 출력
- [x] [사용자] 이름 입력
  - 기존의 유효성 검사 사용
- [x] 이번 달 출석 기록 조회
- [x] 크루원의 이번 달 출석 기록 출력

### 6단계 - 제적 위험자 조회
- [x] 제적 위험자 도메인 정의
- [x] 제적 위험자 조회 로직 구현
- [x] 제적 위험자 리스트 출력
